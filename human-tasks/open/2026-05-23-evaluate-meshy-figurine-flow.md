## Steps

1. Open the Meshy account used for the paid monthly plan.
2. Confirm the API key exists in the Meshy API settings page. Do not paste the key into chat or docs.
3. In Meshy or MakerWorld PrintU, test a small set of inputs:
   - `.tmp/input_image/Profile-Pic-HIMSS.jpg`
   - `.tmp/input_image/Gemini_Generated_Image_lzneejlzneejlzne.png`
   - Any customer-like photo Elliot is comfortable using for this test.
4. Test the PrintU-like choices that matter for our UI: Bobblehead, Chibi, Cartoon, Emoji, Natural pose, Image pose, and T-pose when available.
5. Prioritize the current approved upstream workflow: source photo to Meshy Creative Lab Figure prototype/build to smooth no-base GLB, with posture/style controls exposed only where the API and product copy can support them honestly.
6. For each useful generation, capture safe notes: style, posture, time, credits/cost, output formats available, whether GLB/STL/3MF download works, and whether the model looks sellable.
7. Open the generated STL/3MF in Bambu Studio, OrcaSlicer, or another slicer and note printability issues: scale, supports, fragile parts, color mapping, warnings, and estimated print time/material.
8. Open Experiment 010 outputs under `.tmp/experiments/meshy/standard/exp-010-creative-lab-print-tools`.
9. In Blender/slicer, compare original GLB, textureless repaired watertight GLB, textured remeshed GLB/STL/3MF, visual smoothness, underside/feet geometry, scale, contact area, support needs, and fit to the deterministic `printu-round-v1` base.
10. For the current square-base path, import `.tmp/print-files/N6wSBUfLdEcQy82BG3l1duHmXTY2/f604d393-bfa2-4779-b05b-f6a2082604c9/figurine/creative-lab-original/model.glb` and `.tmp/gold-standard/Figurine Standard Square Base/full_color/base.glb` into a clean Blender scene. They should match scale without resizing.
11. In slicer/print-review planning, use `150mm` figurine height as the target. The matching package should be about `62.14mm x 54.42mm x 150.00mm` for the body and `105.24mm x 105.24mm x 24.00mm` for the square base before final supports/repair decisions.
12. Cloudflare-backed HTTPS receiver is created and smoke-tested. Use payload URL `https://api.3dprintyou.com/webhooks/meshy`.
13. `MESHY_WEBHOOK_SECRET` is present in local `.env` and uploaded as an encrypted Cloudflare Worker secret. A real Meshy delivery confirmed the secret arrives in `x-meshy-api-webhook-secret-key`, and the Worker now rejects webhook POSTs without the matching secret.
14. Meshy API settings webhook is active and delivered real `PENDING` and `FAILED` events to the Worker.
15. For future Cloudflare automation, expand or replace the local Cloudflare API token. On 2026-05-23 the current root `.env` token verified successfully and could see both project zones, but returned `403` for DNS record and Worker route reads.
